# 🚀 Quick Deploy to Linux - Platform-Specific Guides\n\nDeploy your AI Cloud Cost Detective to popular Linux hosting platforms in minutes.\n\n---\n\n## 🏆 Recommended: DigitalOcean ($5/month)\n\n### Step 1: Create Droplet\n1. Go to https://digitalocean.com\n2. Click \"Create\" → \"Droplet\"\n3. Choose:\n   - **Image**: Ubuntu 22.04 LTS\n   - **Size**: Basic $5/month (1 GB RAM, 25 GB SSD)\n   - **Region**: Choose closest to you\n   - **Authentication**: SSH key (or password)\n4. Click \"Create Droplet\"\n\n### Step 2: Connect & Setup\n```bash\n# SSH into your droplet\nssh root@your-droplet-ip\n\n# Follow: LINUX_SERVER_SETUP.md Part 1-4\n```\n\n### Step 3: Deploy\n```bash\ncd /root\ngit clone https://github.com/yourusername/AI-Cloud-Cost-Detective.git\ncd AI-Cloud-Cost-Detective\n\n# Create .env\ncat > .env << EOF\nOPENAI_API_KEY=your_key\nJWT_SECRET=your_secret\nEOF\n\n# Start\ndocker-compose up -d\n```\n\n**Access at**: `http://your-droplet-ip:3000`\n\n**Cost**: $5/month\n\n---\n\n## 🐙 Alternative: AWS EC2 (Free for 12 months)\n\n### Step 1: Launch Instance\n1. Go to https://aws.amazon.com/ec2\n2. Click \"Launch Instance\"\n3. Choose:\n   - **AMI**: Ubuntu 22.04 LTS\n   - **Instance type**: t3.micro (free tier)\n   - **Storage**: 20 GB (free tier)\n4. Click \"Launch\"\n5. Download key pair (.pem file)\n\n### Step 2: Connect\n```bash\nchmod 400 your-key.pem\nssh -i your-key.pem ubuntu@your-instance-ip\n\n# Follow: LINUX_SERVER_SETUP.md Part 1-4\n```\n\n### Step 3: Deploy\nSame as DigitalOcean\n\n**Access at**: `http://your-instance-ip:3000`\n\n**Cost**: Free for 12 months (free tier)\n\n---\n\n## 🚂 Alternative: Railway (Free tier available)\n\n### Easiest - No server setup needed!\n\n### Step 1: Push to GitHub\n```bash\ngit push origin main\n```\n\n### Step 2: Connect Railway\n1. Railway auto-deploys\n\n**Access at**: `https://your-project.railway.app`\n\n**Cost**: Free tier ($5/month credit) or $5+/month paid\n\n---\n\n## 🎯 Alternative: Render (Free tier)\n\n### Also very easy!\n\n### Step 1: Create Web Service\n1. Create Account\n```\nhttps://digitalocean.com\nSign up → Verify email\n```\n\n### 2. Create Droplet\n```\nCreate → Droplet\n- Ubuntu 22.04 LTS\n- $5/month (Basic)\n- Your region\n- SSH key (recommended)\nCreate Droplet\n```\n\n### 3. Copy Server IP\n```\nSee your droplet's IP address\nExample: 123.456.789.012\n```\n\n### 4. SSH In\n```bash\nssh root@123.456.789.012\n# Or with key:\nssh -i your-key ubuntu@123.456.789.012\n```\n\n### 5. Install Docker\n```bash\ncurl -fsSL https://get.docker.com -o get-docker.sh\nsudo sh get-docker.sh\nsudo usermod -aG docker $USER\n```\n\n### 6. Clone Project\n```bash\ncd /root\ngit clone https://github.com/yourusername/AI-Cloud-Cost-Detective.git\ncd AI-Cloud-Cost-Detective\n```\n\n### 7. Create .env\n```bash\ncat > .env << EOF\nUSE_LOCALSTACK=true\nOPENAI_API_KEY=sk-proj-your_key\nJWT_SECRET=your_random_secret\nEOF\n```\n\n### 8. Start\n```bash\ndocker-compose up -d\n```\n\n### 9. Access\n```\nhttp://123.456.789.012:3000\n```\n\n**Done! ✅**\n\n---\n\n## 🔧 Common Issues\n\n### Port 80/443 blocked\n```bash\n# Use different port\ndocker-compose.yml:\nprovider:\n  ports:\n    - \"8080:3000\"  # Use 8080 instead\n```\n\n### Out of memory\n```bash\n# Increase swap\ndd if=/dev/zero of=/swapfile bs=1G count=2\nchmod 600 /swapfile\nmkswap /swapfile\nswapon /swapfile\n```\n\n### Container keeps restarting\n```bash\n# Check logs\ndocker-compose logs backend\n\n# Check .env variables\ncat .env\n```\n\n### Can't access from browser\n```bash\n# Check firewall\nsudo ufw status\n\n# Allow ports\nsudo ufw allow 80\nsudo ufw allow 443\nsudo ufw allow 3000\nsudo ufw allow 8000\n```\n\n---\n\n## 📋 Deployment Checklist\n\n### Before Deployment\n- [ ] Code pushed to GitHub\n- [ ] Dockerfile created (for backend & frontend)\n- [ ] docker-compose.yml configured\n- [ ] .env file created with secrets\n\n### During Deployment\n- [ ] Server created (Ubuntu 22.04)\n- [ ] SSH access verified\n- [ ] Docker installed\n- [ ] Project cloned\n- [ ] .env file created\n- [ ] `docker-compose up -d` running\n\n### After Deployment\n- [ ] All containers running: `docker-compose ps`\n- [ ] Frontend accessible: `http://ip:3000`\n- [ ] Backend API works: `http://ip:8000/docs`\n- [ ] Can signup/login\n- [ ] Can run analysis\n\n---\n\n## 🌍 Custom Domain (Optional)\n\n### Point Domain to Server\n\n1. **Buy domain** (Namecheap, GoDaddy, Route53, etc.)\n\n2. **Get DNS settings** from hosting:\n   - DigitalOcean: Create \"A\" record\n   - AWS: Use Route53\n   - Linode: Use Linode DNS\n\n3. **Update domain DNS**:\n   ```\n   A record: @ → your-server-ip\n   CNAME: www → your-domain.com\n   ```\n\n4. **Setup SSL** (Let's Encrypt):\n   ```bash\n   sudo apt install certbot\n   sudo certbot certonly --standalone -d your-domain.com\n   ```\n\n5. **Update Nginx** config to use domain\n\n---\n\n## 💡 Recommended Path\n\n1. **Start with LocalStack** (your laptop)\n   -
