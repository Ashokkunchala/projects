# AI Agent Platform Brain — 5-Phase Implementation Plan

## Goal
Transform the AI Agent from a standalone IaC analysis page into the **central intelligence layer** that drives every feature in the platform — scan orchestration, result interpretation, cost optimization, infrastructure visualization, and free tier management.

---

## Current State Summary

**Cloudflare Worker** (`infrastructure/cloudflare/src/index.ts`, 673 lines):
- 5 endpoints: `/analyze`, `/validate`, `/explain`, `/complete`, `/health`
- Uses `@cf/meta/llama-3.2-3b-instruct` (3B — too small for complex reasoning)
- D1 for analysis history, KV for rate limiting (10 req/min/IP), R2 for storage
- No conversation/chat support, no streaming, no user context

**Backend cloudflare_ai.py** (241 lines):
- 9 helper functions that call Worker's `/complete` endpoint via `httpx`
- Synchronous `httpx.Client` calls (blocking, not async)
- All return `None` on failure — graceful degradation but no retry logic
- No conversation history, no context injection

**Frontend AIAgent.tsx** (216 lines):
- Standalone page — paste code, pick action, get JSON result
- Raw JSON output (no markdown rendering)
- No chat interface, no follow-up questions, no context from other pages

**Backend routers using cloudflare_ai.py**:
- `routers/cost.py` — calls `cost_spike_explanation`, `forecast_narrative`, `rightsizing_explanation`, `personalized_awareness`
- `routers/infra.py` — calls `infra_diagram_summary`, `infra_validation_analysis`
- `routers/free_tier.py` — calls `free_tier_recommendations`, `free_tier_eligibility_check`
- `routers/agent.py` — pure proxy to Worker's `/analyze`, `/validate`, `/explain`, `/complete`

**Backend main.py** (1327 lines):
- `_run_analysis()` — the scan background task that calls `ai_analyzer.analyze_resources()`
- After scan completes, stores result in DB — no AI summary generated automatically
- `ai_analyzer.py` — supports 14 AI providers with auto-detection, falls back to rule engine

**Database schema** (PostgreSQL):
- `users`, `analyses`, `revoked_tokens`, `password_reset_tokens`, `scheduled_scans`
- No conversations table, no chat messages table

**D1 schema** (`infrastructure/cloudflare/schema.sql`):
- `analyses`, `users`, `scan_history`, `free_tier_usage`, `infra_diagrams`, `cost_estimates`, `agent_cache`
- No conversations or messages tables

## Phase 1: Upgrade Model + Chat Endpoint + Conversation Storage

### 1.1 Upgrade AI Model in Worker

**File**: `infrastructure/cloudflare/src/index.ts`

- Change `@cf/meta/llama-3.2-3b-instruct` → `@cf/meta/llama-3.1-8b-instruct` in all `aiBinding.run()` calls (lines 188, 280, 348, 400, 635)
- The 8B model is significantly better at structured JSON output, multi-step reasoning, and consistent responses

### 1.2 Add Chat Endpoint to Worker

**File**: `infrastructure/cloudflare/src/index.ts`

Add new endpoint `POST /api/agent/chat`:

```
Request: {
  messages: Array<{ role: 'user'|'assistant'|'system', content: string }>,
  context?: {
    analysis_id?: string,
    analysis_result?: object,
    scan_data?: object,
    page?: string,
  },
  max_tokens?: number,
  temperature?: number,
}
```

The endpoint:
1. Takes the conversation messages array + optional context
2. Builds a system prompt that includes:
   - Role: "You are the AI Cost Detective assistant..."
   - Context injection: if `analysis_result` is provided, inject a summary of the scan results
   - If `page` is provided, tailor behavior (e.g., on Report page, focus on explaining issues; on Dashboard, focus on overview)
3. Calls `@cf/meta/llama-3.1-8b-instruct` with the full message history
4. Returns `{ response: string, suggestions?: string[] }`

**Streaming support**: Use Cloudflare Workers' `TransformStream` to stream the AI response token-by-token. The Worker supports SSE (Server-Sent Events) — return `Content-Type: text/event-stream` with `data: {token}` lines.

## Phase 4: Model Router (Cloudflare Simple / Claude-GPT Complex)

### 4.1 Complexity Classifier

**File**: `backend/cloudflare_ai.py`

Add function `classify_complexity(prompt: str) -> str`:
- Simple: single resource questions, "what is X", basic explanations → route to Cloudflare Worker (free)
- Complex: multi-resource analysis, architectural decisions, code generation, compliance checks → route to Claude/GPT (paid, higher quality)

Classification heuristics:
- Prompt length > 500 tokens → likely complex
- Keywords: "compare", "architecture", "generate", "terraform", "compliance", "audit" → complex
- Keywords: "explain", "what is", "help", "how to" → simple
- If analysis_result context is provided with > 50 resources → complex

### 4.2 Unified Router Function

**File**: `backend/cloudflare_ai.py`

Replace all individual functions with a unified dispatch:
```python
async def agent_chat(messages: list[dict], context: dict = None, force_provider: str = None) -> str:
    complexity = classify_complexity(messages[-1]["content"])
    
    if force_provider:
        provider = force_provider
    elif complexity == "complex" and _has_paid_provider():
        provider = "paid"  # Claude/GPT via ai_analyzer.py
    else:
        provider = "cloudflare"  # Free tier via Worker
    
    if provider == "cloudflare":
        return await _cloudflare_chat(messages, context)
    else:
        return await _paid_chat(messages, context)
```

### 4.3 Paid Provider Chat Function

**File**: `backend/cloudflare_ai.py`

Add `_paid_chat()` that uses the existing `ai_analyzer.py` provider infrastructure:
- Import the provider registry from `ai_analyzer._PROVIDER_REGISTRY`
- Send the full conversation as a single prompt (paid providers handle multi-turn differently)
- Use the highest-quality model available (Claude > GPT-4o > Gemini > others)

### 4.4 User Provider Preference

**File**: `frontend/src/pages/Dashboard.tsx` and `frontend/src/api.ts`

Add AI provider selector to the settings area (already exists for scan analysis). Extend it to also control the chat Agent's model:
- "Auto" (default) — router decides based on complexity
- "Cloudflare AI (Free)" — force free tier
- "Claude / GPT-4o" — force paid for best quality

## Phase 5: Advanced Agent Capabilities

### 5.1 Resource Lookup Tool

**File**: `backend/routers/agent.py`

Add `POST /api/agent/resource-lookup`:
- Takes a natural language query like "show me all stopped EC2 instances"
- The Agent parses the query, queries the latest analysis result from DB, filters resources
- Returns structured results + Agent explanation

Implementation: The Worker sends a prompt to Llama asking it to extract filter criteria (service, state, type) from the natural language query. Backend applies those filters to the stored analysis data.

### 5.2 Fix Generator

**File**: `backend/cloudflare_ai.py`

Add `generate_fix(issue: dict, context: dict) -> dict`:
- Takes a specific issue from scan results
- Generates a complete, copy-paste-ready fix command
- Includes prerequisites ("make sure you have IAM permissions for...")
- Includes rollback plan ("to undo: ...")
- Returns `{ command: str, explanation: str, prerequisites: str[], rollback: str }`

### 5.3 Cost Simulator

**File**: `backend/cloudflare_ai.py`

Add `simulate_cost_change(resource: dict, proposed_change: str) -> dict`:
- "What if I downsize m5.xlarge to m5.large?"
- Agent calculates estimated savings based on AWS pricing
- Considers implications (CPU/memory tradeoffs, impact on performance)
- Returns `{ current_cost: float, new_cost: float, savings: float, risks: str[], recommendation: str }`

### 5.4 IaC Generator

**File**: `backend/routers/agent.py`

Add `POST /api/agent/generate-iac`:
- Takes a description: "Create a VPC with 2 public subnets, an ALB, and 2 EC2 instances"
- Agent generates Terraform or CloudFormation code
- Returns `{ code: str, format: 'terraform'|'cloudformation', explanation: str }`

### 5.5 Compliance Checker

**File**: `backend/cloudflare_ai.py`

Add `compliance_check(resources: dict, framework: str = 'cis') -> dict`:
- Analyzes scan results against CIS AWS Foundations Benchmark or other frameworks
- Returns `{ score: int, violations: list, recommendations: list, passed_checks: int, total_checks: int }`

### 5.6 Anomaly Detection

**File**: `backend/cloudflare_ai.py`

Add `detect_anomalies(current_scan: dict, historical_scans: list) -> list`:
- Compares current scan with historical data
- Detects: new resources added, resources removed, cost changes, new issue types
- Returns list of anomalies with severity and explanation

### 5.7 Frontend Integration for Advanced Features

**File**: `frontend/src/components/ChatWidget.tsx`

Add quick-action buttons in the chat interface:
- "Analyze my scan" — sends analysis context
- "Suggest fixes" — triggers fix generator for all HIGH issues
- "Simulate change" — opens a dialog to describe a proposed change
- "Check compliance" — runs compliance check against CIS
- "Generate Terraform" — opens IaC generator flow

**File**: `frontend/src/pages/AIAgent.tsx`

Redesign the AI Agent page:
- Replace the simple textarea + result layout with a full chat interface
- Add tabs: Chat | Analyze | Validate | Explain (keeping old functionality but in a chat wrapper)
- Show conversation history in a sidebar
- Allow switching between the old JSON view and the new chat view
